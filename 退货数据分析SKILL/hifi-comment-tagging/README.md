# HiFi 退货与评论售后分析打标 Skill (Hifi Comment Tagging)

</div>

聚焦于针对 HiFi 音频类主打相关外设设备（如 P4、ZD3、ZA3、LC30 等）反馈数据的专项梳理套件。
用于对接未整序过的表格反馈堆栏，自动清洗噪音数据并实施规范的**四级分级打标（一级至四级归因）**，最终直出成可供产品经理参考的“归总摘要 + 高度格式化数据”的工作簿。

## 🌟 核心能力与适用场景

当您需要对以下情况进行复盘分析时，请引用此应用模块：
- **目标单一归拢分析**：针对一个核心主体目标（通常为某具体型号：如 P4 / ZA3）提取用户的售前售后问题清单进行排查整理。
- **自动排废脱水**：清除无用客诉空白信息包括：“同上”、“NA”、或其他无意义信息。并在有中文翻译时优先提取中文表述辅助解读。
- **标准化模式嵌套**：无需从零总结，沿用内定已有的《问题点：四级细分映射》进行强制落地：即归拢进入`一级分类 -> 二级分类 -> 三级问题点 -> 四级归因` 标准链路。
- **自动总结排版输出**：提供基于 PM 视角的数据总结文档（包括总计、TOP3问题排序占比、重要问题枚举）以及生成限制数量和内含样式的图表呈现。

## 🚀 工作流程与核心脚本

该方案必须以**严格的时序方式**被处理或调用：
1. **初筛排查验证** (`profile_workbook.py`): 对提交的大规模 Excel 工作薄验证内部页面与抬头位置。必须且只能挑选确认针对**唯一一个**产品。 
2. **清洗净化** (`clean_product_comments.py`): 重构或废弃非相关产品及排查无效语句。支持选项 `--keep-blank` 防止特殊情况需求下必须保留全量（哪怕没评价内容）反馈。
3. **标签与模板融合建立** (`extract_taxonomy_examples.py` & `build_summary_scaffold.py`): 基于现有积累及大纲实施关联判断挂靠。遇无定论类别将被特别分离标有 `待人工判定 / 暂无法确定` 的预警标记以便后续补交核查。生成具备层级、色块标识和最终结果总述摘要（Summary）在内的结果文档。

## 🔔 注意提示
1. 涉及所有 `待人工判定` 或是 `待人工筛查` 的非标准类疑难反馈结果，将**禁止**静默收录至最终执行表内进行错误计算。它会第一时间抛出交由操作者（或者用户）在会话中手动引导补充决定后，才作数终结处理。
2. 总结文本仅限在 Excel 文件生成，将保持产品经理正常专业报告的长文本连贯陈述样式。
